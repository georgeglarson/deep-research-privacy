Quick Start Guide

Get started with Deep Research's core functionality

1

Core Setup

Get Your API Keys

  1. Visit Venice.ai
    • Sign up for an account
    • Navigate to API settings
    • Generate a new API key
  2. Visit Brave Search API
    • Create a developer account
    • Visit API Keys section
    • Generate a new API key
2

Installation

bash 
# Clone the repository
git clone https://github.com/georgeglarson/deep-research-privacy.git
cd deep-research-privacy

# Copy example config
cp .env.example .env

Edit .env with your API keys:

env 
VENICE_API_KEY=your_venice_api_key_here
BRAVE_API_KEY=your_brave_search_api_key_here
3

Basic Usage

Quick Start

bash 
# Install dependencies
npm install

# Start Deep Research
npm start

You'll be prompted to:

  • Enter your research topic
  • Choose research breadth (2-10, default: 3)
  • Choose research depth (1-5, default: 2)

Advanced Usage

You can also specify parameters directly in the command:

bash 
# Specify all parameters in one command
npm start -- research "quantum computing" --breadth 3 --depth 2

Understanding Parameters:

  • Breadth (2-10): Number of sources to analyze
    • Default: 3
    • Higher values = more diverse sources
  • Depth (1-5): How deeply to analyze each source
    • Default: 2
    • Higher values = more detailed analysis
  • Model: AI model to use
    • Default: deepseek-r1-671b (best for deep research)
    • llama-3.2-3b (fastest)
    • llama-3.1-405b (most intelligent)
    • dolphin-2.9.2-qwen2-72b (most uncensored)
    • Or specify any model name directly
bash 
# Use a specific model
npm start -- research "quantum computing" --model llama-3.1-405b

Results are automatically saved to the 'research' directory as markdown files.

Real-time Monitoring (Optional)

For detailed progress visualization, use two terminal windows:

bash 
# Terminal 1: Start the visualization UI
npm run start:ui

# Terminal 2: Add --ui flag to your research
npm start -- research "your topic" --breadth 3 --depth 2 --ui

The UI dashboard provides:

  • Real-time progress tracking
  • Live API call monitoring
  • Search result visualization
  • Knowledge synthesis updates
4

Example Research

Here are some example research queries with different parameters:

bash 
# Broad overview (many sources, quick analysis)
npm start -- research "AI ethics" --breadth 5 --depth 1

# Deep dive (fewer sources, thorough analysis)
npm start -- research "quantum computing basics" --breadth 2 --depth 3

# Balanced research (recommended for most topics)
npm start -- research "climate change solutions" --breadth 3 --depth 2

Each approach yields different insights:

  • High breadth (4-10): Best for new topics or getting multiple perspectives
  • High depth (3-5): Ideal for technical subjects requiring detailed understanding
  • Balanced (3,2): Good default for most research needs

Tip: Add the --ui flag to any command to enable real-time visualization
